# Security Policy

## Supported Versions

Qinnovate is a standards body. Security issues in the standards themselves should be reported.

| Version | Supported          |
| ------- | ------------------ |
| QIF 3.x (Hourglass) | :white_check_mark: |
| QIF 2.x | :x: (deprecated) |
| ONI 14-layer | :x: (legacy) |

## Reporting a Vulnerability

If you discover a security vulnerability in the QIF specification or governance standards, please report it responsibly:

**Email:** security@qinnovate.com

**Please include:**
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact on BCI security
- Suggested remediation (if any)

**Response Time:**
- Acknowledgment: Within 48 hours
- Initial assessment: Within 7 days
- Resolution timeline: Depends on severity

## Security Standards

Qinnovate maintains security through:

1. **Peer Review** — All standard changes reviewed by domain experts
2. **Transparent Governance** — Public review process for major changes
3. **Version Control** — All changes tracked in git with full audit trail
4. **CI/CD Security** — Automated security scanning on all commits
5. **Dependency Scanning** — Dependabot monitors for vulnerable dependencies

## Disclosure Policy

- **Private disclosure period:** 90 days for critical issues
- **Public disclosure:** After patch or mitigation is available
- **Credit:** Reporters will be credited (unless they prefer anonymity)
